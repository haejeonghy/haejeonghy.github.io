## Amazon VPC(Virtual Private Cloud)란 무엇인가요?

간단히 말하면, Amazon VPC는 **AWS 내에서 내가 제어할 수 있는 가상의 네트워크 공간**입니다. 마치 회사 사무실 건물 안에서 방을 나눠 쓰듯이, 클라우드 안에 원하는 네트워크 구조를 만들어 자원을 배치할 수 있게 해줍니다.

공식 설명을 요약하면 다음과 같습니다.

> 정의한 논리적으로 격리된 가상 네트워크에서 AWS 리소스를 시작할 수 있습니다.

> 이 가상 네트워크는 AWS의 확장 가능한 인프라를 사용하면서도, 자체 데이터센터 네트워크와 유사하게 구성할 수 있습니다.

## 왜 중요하고 어떤 장점이 있나요?

- 내 데이터센터처럼 네트워크를 설계할 수 있어 **제어력**이 높습니다.
  - IP 주소 범위 설정, 서브넷 배치, 라우팅, 게이트웨이 연결 등
- AWS 리전 안에서 격리된 환경을 제공해 **보안 및 네트워크 분리**에 유리합니다.
- 온프레미스나 다른 클라우드 네트워크와 연결해 **하이브리드 아키텍처**로 확장할 수 있습니다.
- AWS 인프라 위에서 동작하므로 물리 네트워크 장비를 직접 관리하지 않아도 됩니다.

## 카페 시스템 예시

카페를 AWS 위에서 운영한다고 가정해보겠습니다.

- 카페 건물 전체가 VPC입니다.
- 건물 안의 여러 방이 서브넷입니다.
  - 예: 주방, 고객 대기실, 관리자 사무실
- 각 방에는 EC2, DB 같은 자원이 배치됩니다.
- 고객 대기실은 인터넷 게이트웨이로 외부와 연결해 웹 사용자가 접근하게 할 수 있습니다.
- 관리자 사무실은 외부 인터넷 직접 접근을 차단하고 내부 사용자만 접근하게 할 수 있습니다.
- 본사(온프레미스)나 다른 지점(VPC)과는 VPN, 피어링으로 안전하게 연결할 수 있습니다.

즉, VPC를 통해 **구역을 나누고**, **외부 접근을 통제하며**, **내부 통신을 안전하게 설계**할 수 있습니다.

## 핵심 구성 요소

- **서브넷(Subnet)**: VPC 내부의 더 작은 네트워크 단위
- **IP 주소 지정**: VPC/서브넷에 IPv4, IPv6 대역 할당
- **라우팅 테이블(Routing Table)**: 트래픽 경로를 결정하는 규칙 집합
- **게이트웨이/엔드포인트**
  - 인터넷 게이트웨이(Internet Gateway): 인터넷 연결 출입구
  - VPC 엔드포인트(VPC Endpoint): 인터넷 없이 AWS 서비스와 통신
- **VPC 피어링(VPC Peering)**: 서로 다른 VPC 간 직접 통신
- **VPN/Transit Gateway**: 온프레미스 및 다중 VPC를 확장 연결

## VPC가 없을 때의 불편사항

### 1. 네트워크 격리 부족으로 보안 리스크 증가

- 리소스 간 접근 통로를 세밀하게 분리/통제하기 어려워집니다.
- 외부 침입이나 비의도적 노출 위험이 커질 수 있습니다.

카페 비유:
방 구분이 없는 공유 공간이라면, 손님이 주방이나 재고 창고에 접근할 위험이 커집니다.

### 2. 서브넷·라우팅 제어 불가로 구조 설계 어려움

- 서비스 특성에 맞춘 네트워크 토폴로지 설계가 어렵습니다.
- 공개/사설 영역 분리, 라우팅 정책 최적화가 제한됩니다.

카페 비유:
고객 대기실, 주방, 재고실을 분리하고 싶어도 구조적으로 나눌 수 없는 상황과 같습니다.

### 3. 외부 인터넷·온프레미스 연계 복잡

- 내부 자원을 외부에 노출하지 않고 안전하게 연동하기 어려워질 수 있습니다.
- 하이브리드 연결(VPN, 피어링) 설계 자유도가 낮아집니다.

카페 비유:
본사와 지점을 내부망으로 안전하게 연결해야 하는데, 외부 복도만 거쳐야 하는 상황이 됩니다.

### 4. 확장성과 유연성 저하

- 초기에는 단순해 보여도, 서비스가 커지면 네트워크 고도화가 어렵습니다.
- 보안/운영 복잡도가 뒤늦게 급격히 증가할 수 있습니다.

카페 비유:
고객과 메뉴가 늘어나도 공간 분리가 안 되면 운영 동선과 통제가 급격히 어려워집니다.

### 5. 규제·감사 대응 난이도 증가

- 금융/보험/마이데이터처럼 민감 정보가 있는 환경에서 격리와 추적이 중요합니다.
- 접근 제어와 트래픽 흐름 추적 체계가 부족하면 감사 대응이 어려워집니다.

카페 비유:
결제/관리 구역 구분과 출입 기록이 없으면, 사고 발생 시 원인 추적이 매우 어려워집니다.
