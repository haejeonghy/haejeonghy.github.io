### **– 리소스 사용량 조회**

- kubectl top 명령은 **메트릭 서버(Metrics Server)**가 설치되어 있어야 동작함
- **현재 노드와 파드의 CPU/메모리 사용량을 실시간 확인** 가능

### **📌 작동 원리**

- 메트릭 서버가 kubectl top 요청을 수신
- API 서버를 통해 각 노드에 실행 중인 kubelet으로 전달
- kubelet은 내장된 cAdvisor를 통해 CPU/메모리/네트워크/디스크 등의 자원 사용량을 수집
- 외부 모니터링 솔루션들도 cAdvisor의 /metrics/cadvisor 엔드포인트에서 메트릭 수집 가능

### **📌 예시 명령어**

```
kubectl top pod -A --sort-by=memory
```

### **🔹 2. df-pv**

![df-pv설치.png](/resource/post-format/16%EC%9E%A5-%EA%B0%84%EB%8B%A8%ED%95%98%EA%B2%8C-%EC%82%AC%EC%9A%A9%ED%95%A0-%EC%88%98-%EC%9E%88%EB%8A%94-%EC%BF%A0%EB%B2%84%EB%84%A4%ED%8B%B0%EC%8A%A4-%EB%AA%A8%EB%8B%88%ED%84%B0%EB%A7%81-%EB%8F%84%EA%B5%AC/df-pv%E1%84%89%E1%85%A5%E1%86%AF%E1%84%8E%E1%85%B5-1.png)

### **– 볼륨 사용량 확인 도구**

- PVC(Persistent Volume Claim)의 **디스크 사용량을 시각적으로 확인** 가능
- kubectl krew install df-pv로 설치
- 터미널 기반으로 각 파드의 PVC 사용 현황을 표로 출력함

### **– 실시간 쿠버네티스 관리 도구**

- TUI(터미널 UI) 기반 **인터랙티브 쿠버네티스 리소스 관리 도구**
- 파드, 디플로이먼트, 노드, 서비스 등 다양한 리소스 실시간 조회 및 조작 가능

### **주요 기능**

| **기능** | **설명** |
| --- | --- |
| / | 리소스 검색 |
| l | 로그 보기 |
| d | 리소스 삭제 |
| s | 리소스 상세 |
| : | 명령 실행 |
| :q | 종료 |
