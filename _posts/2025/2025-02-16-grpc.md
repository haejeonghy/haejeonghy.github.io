## 📌 gRPC란?

**gRPC**는 `HTTP/2` 기반에서 `Protocol Buffers (protobuf)`를 사용하여

**직렬화된 바이트 스트림**으로 통신하는 RPC(Remote Procedure Call) 프레임워크입니다.

📌 **JSON 기반 통신보다 가볍고 빠른 이유:**

✅ `protobuf` 직렬화 → **더 작은 패킷 크기**

✅ `HTTP/2` 멀티플렉싱 → **동시에 여러 요청 처리 가능**

✅ **MSA(Microservices Architecture)에서 Internal API 통신 최적화**

---

## 🚀 gRPC의 주요 장점

✅ **Latency 감소** (더 빠른 응답)

✅ **더 많은 트래픽 처리 가능** (고성능)

✅ **HTTP/2 기반** (멀티플렉싱 지원)

✅ **다양한 언어 지원** (Polyglot 환경에서 유용)

✅ **Protocol Buffers 사용** (빠르고 효율적인 데이터 직렬화)

---

## 🔄 gRPC 통신 방식

### **1️⃣ Unary RPC (단건 요청/응답)**

📌 **Unary** = "ping-pong" 같은 단건의 요청/응답 구조

- `binary(2진법) ↔ unary(1진법)` 개념에서 유래
- gRPC에서는 하나의 요청을 보내고 하나의 응답을 받는 방식

### **2️⃣ Streaming RPC**

📌 **Unary 방식과 반대로, 여러 개의 데이터 스트림을 주고받는 방식**

- **Server Streaming** → 클라이언트가 요청하면 서버가 여러 개의 응답을 반환
- **Client Streaming** → 클라이언트가 여러 개의 데이터를 보내고 서버가 응답
- **Bidirectional Streaming** → 서버와 클라이언트가 동시에 여러 개의 데이터를 주고받음

---

## 🔧 **WithUnaryServerChain** (여러 개의 인터셉터 적용)

📌 gRPC 서버는 인터셉터를 받을 수 있지만, 기본적으로 하나만 가능

📌 여러 개의 인터셉터를 연결하고 싶다면 `WithUnaryServerChain`을 사용하여 체인처럼 연결 가능

---

## 🛠️ **Register Service (서비스 등록)**

📌 **gRPC 서비스를 gRPC 서버에 등록하는 과정**

📌 gRPC에서 서비스를 정의하면 서버에 **직접 등록해야 호출 가능**
