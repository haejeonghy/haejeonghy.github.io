# 🔁 `sync.Once` 개념 및 사용 정리

## 📌 개요

`sync.Once`는 Go의 동시성 제어 도구로, **특정 함수를 딱 한 번만 실행하도록 보장**하는 구조체이다.

- 여러 고루틴이 동시에 호출해도, 지정된 함수는 **오직 한 번만 실행**
- 주로 **초기화 코드**에 사용됨

---

## 🧾 사용 예시

```go
package main

import (
    "fmt"
    "sync"
)

var once sync.Once

func initialize() {
    fmt.Println("초기화 코드 실행됨")
}

func main() {
    for i := 0; i < 3; i++ {
        go func() {
            once.Do(initialize)
        }()
    }

    time.Sleep(1 * time.Second)
}

```

### ✅ 결과

```
초기화 코드 실행됨

```

- 여러 고루틴이 호출해도 `initialize()`는 **한 번만 실행**

---

## ⚙️ 주요 특징

| 항목 | 설명 |
| --- | --- |
| **한 번만 실행** | `Do(f)` 호출 시, `f()`는 최초 한 번만 실행됨 |
| **goroutine-safe** | 내부적으로 락을 사용하여 동시 접근에도 안전 |
| **재사용 불가** | 한 번 실행된 `Once`는 다시 초기화할 수 없음 |
| **panic 발생 시 재실행** | `f()`에서 panic이 나면 실패로 간주되어 이후 `Do()` 호출 시 다시 시도됨 |

---

## 🧠 사용 사례

- 설정 파일 로딩
- 로깅 시스템 초기화
- 싱글턴 객체 생성
- 데이터베이스 연결 초기화
- 공통 리소스 준비 작업

---

## ❗ 주의사항

- `Once`는 상태를 내부에 저장하므로 전역 변수로 선언하거나, 구조체 내에 포함시켜 사용해야 함
- `f()` 함수가 오래 걸리면 다른 goroutine은 **block**됨
- 재사용이 불가능하므로 반복적 초기화가 필요한 경우엔 적합하지 않음
