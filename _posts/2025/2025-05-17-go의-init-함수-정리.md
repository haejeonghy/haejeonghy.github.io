# **⚙️ Go의 init() 함수 정리📌**

## **init() 함수란?**

Go 언어에서 init() 함수는 **특수한 예약 함수**로, 다음과 같은 특징을 지닙니다:

- 반환값과 인자를 가질 수 없음
- 패키지가 import될 때 **자동으로 실행**
- 패키지 내 여러 파일, 여러 개의 init()이 존재 가능
- 주로 **패키지 전역 변수 초기화** 등에 사용

```
func init() {
    // 초기화 로직
}
```

---

## **🔄 실행 순서**

1. 의존 패키지의 전역 변수 초기화
2. 의존 패키지의 init() 실행
3. 본 패키지의 전역 변수 초기화
4. 본 패키지의 init() 실행
5. 최종적으로 main() 실행

---

## **🧾 예시 코드**

```
package cafe

var Menu []string

func init() {
    Menu = []string{"아메리카노", "라떼"}
    fmt.Println("카페 메뉴가 초기화되었습니다.")
}
```

이 경우 cafe 패키지를 import하기만 해도 init()이 자동 실행됩니다.

---

## **⚠️ init() 사용을 지양해야 하는 이유**

| **문제점** | **설명** |
| --- | --- |
| ❌ **암묵적 실행** | 호출 위치가 코드상에 드러나지 않아 **추적이 어려움** |
| ❌ **테스트 어려움** | init()은 테스트에서 **명시적으로 호출하거나 우회할 수 없음** |
| ❌ **순서 예측 불가** | 여러 init()이 존재할 경우 **실행 순서가 직관적이지 않음** |
| ❌ **숨겨진 부작용** | import만 해도 실행되므로 **의존성으로 인한 사이드이펙트** 발생 가능 |
| ❌ **의존성 주입과 충돌** | Fx, Wire 등의 **DI 프레임워크와 개념적으로 충돌** |

---

## **✅ 사용을 대신할 대안**

### **1. 명시적 초기화 함수 사용**

```
func InitializeCafe() *Cafe {
    return &Cafe{Menu: []string{"아메리카노", "라떼"}}
}
```

### **2. 의존성 주입 도구 활용 (예: Uber Fx, Google Wire)**

```
fx.Provide(NewCafeMenu)
```

### **3. 호출자가 직접 초기화**

- 테스트 코드에서 초기화 통제가 가능
- 의존 관계를 명시적으로 표현할 수 있음
