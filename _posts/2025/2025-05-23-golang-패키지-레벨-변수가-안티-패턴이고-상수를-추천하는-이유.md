## **📌 Golang에서 패키지 레벨 변수가 안티 패턴인 이유와 상수 사용 권장 이유**

### **1. 패키지 레벨 변수란?**

- 함수 외부, 패키지 최상단에 선언된 변수
- 예시:

```
var Env = "production"
```

---

### **2. 패키지 레벨 변수가 안티 패턴인 이유**

### **❗ 1) 전역 상태는 예측을 어렵게 만듦**

- 여러 곳에서 값을 수정할 수 있어, 디버깅이 어려움
- 누가 언제 바꿨는지 추적하기 힘듦

### **❗ 2) 테스트 간 간섭 발생**

- 테스트 중 값이 변경되면, 다른 테스트에 영향을 줄 수 있음

```
var Timeout = 5

func TestSomething(t *testing.T) {
  Timeout = 1 // 이 변경이 전체 테스트에 영향
}
```

### **❗ 3) 의존성이 숨겨짐**

- 함수 외부에서 전역 변수에 의존할 경우, 해당 함수의 동작 조건을 코드만 보고 알기 어려움

### **❗ 4) 초기화 순서 문제**

- 여러 변수, init() 함수 등이 얽혀 있을 때 의도치 않은 동작이 발생할 수 있음

---

### **3. 상수(const) 사용의 장점**

### **✅ 1) 값이 불변 → 안정성과 예측 가능성 확보**

### **✅ 2) 동시성 환경에서도 안전 (읽기 전용)**

### **✅ 3) 매직 넘버/문자열을 방지**

```
const RoleAdmin = "admin"
```

---

### **4. 좋은 예시**

### **❌ 나쁜 예**

```
var Env = "production"
```

### **✅ 좋은 예**

```
const EnvProduction = "production"
```

또는 설정 구조체를 사용하는 방법도 권장:

```
type Config struct {
  Env string
}

func NewConfig() *Config {
  return &Config{Env: "production"}
}
```
