---

## 📌 Ingress란?

**Ingress**는 **클러스터 외부에서 내부의 하나 이상의 서비스로 HTTP 및 HTTPS 트래픽을 관리하는 방법**을 제공합니다.

이를 통해 외부에서 쿠버네티스 클러스터 내의 서비스에 접근할 수 있는 규칙을 설정할 수 있습니다.

📌 **Ingress를 사용하는 이유**

✅ **클러스터 외부에서 내부 서비스로 트래픽 전달 가능**

✅ **특정 호스트명을 통해 서비스 연결 가능**

✅ **쿠버네티스 내부 구조를 몰라도 외부에서 접근 가능**

✅ **트래픽을 효율적으로 라우팅하여 관리 가능**

---

## 🌍 Ingress 활용 사례

### **1️⃣ 웹 애플리케이션을 Kubernetes에서 실행하는 경우**

📌 예를 들어, 여러분의 **웹 애플리케이션이 Kubernetes 클러스터에서 실행 중**이라고 가정해 보세요.

- **Ingress를 설정하면 특정 도메인(예: `www.example.com`)을 통해 들어오는 트래픽을 특정 서비스로 연결 가능**
- **클러스터 외부의 사용자가 내부 구조를 몰라도 웹사이트에 접근 가능**

### **2️⃣ 여러 개의 호스트 이름을 관리하는 경우**

📌 여러 개의 서비스 또는 도메인을 Kubernetes에서 운영하는 경우,

- **Ingress를 사용하면 도메인별로 트래픽을 특정 서비스로 분배할 수 있음**
- **하나의 Ingress 리소스에서 여러 개의 호스트를 관리 가능**

---

## 🔧 Ingress의 주요 기능

✅ **호스트 기반 라우팅** → 특정 도메인(예: `www.example.com`)을 서비스에 매핑

✅ **경로 기반 라우팅** → 특정 URL 패턴을 다른 서비스로 분배

✅ **TLS 지원** → HTTPS 트래픽을 위한 보안 인증서 적용 가능

✅ **로드 밸런싱** → 클러스터 내의 여러 서비스에 대한 트래픽 부하 분산 가능

---

## 🛠️ Ingress 설정 주요 요소

📌 **Ingress 클래스** → 내부 또는 외부 Ingress 컨트롤러 지정 가능

📌 **서비스 업스트림 활성화** → 실제 서비스로만 트래픽 전달

📌 **URL 재작성 (Rewrite Target)** → 특정 경로를 서비스로 변경하여 전달

📌 **네임스페이스 및 레이블 지정** → 특정 애플리케이션 및 환경에 맞게 설정
