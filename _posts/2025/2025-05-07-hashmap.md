### **📝 Golang HashMap 정리**

---

### **1. HashMap 기본 개념**

- *키-값 쌍 (Key-Value Pair)**로 데이터를 저장하는 자료구조
- 데이터를 **O(1)**의 시간 복잡도로 빠르게 조회, 추가, 삭제 가능
- Golang에서는 **map**으로 구현됨

---

### **2. HashMap의 주요 특징**

- **빠른 조회 속도**
- **키**는 **유일**해야 하며, **중복**을 허용하지 않음
- **값**은 중복 가능
- 순서가 없는 비순차형 데이터 구조
- **키**는 **Hashable** 타입이어야 함 (예: int, string, struct 등)
- 동적으로 크기 조절 가능

---

### **3. Golang의 map 기본 사용법**

```go
package main

import "fmt"

func main() {
    // 맵 생성
    ages := make(map[string]int)

    // 데이터 추가
    ages["Alice"] = 25
    ages["Bob"] = 30

    // 데이터 조회
    fmt.Println("Alice의 나이:", ages["Alice"]) // 25
    fmt.Println("Bob의 나이:", ages["Bob"])     // 30

    // 데이터 삭제
    delete(ages, "Alice")
    fmt.Println("Alice 삭제 후:", ages)

    // 데이터 존재 여부 확인
    age, exists := ages["Bob"]
    fmt.Println("Bob 존재 여부:", exists, ", 나이:", age) // true, 30

    // 없는 키 접근 (기본값)
    fmt.Println("Charlie의 나이:", ages["Charlie"]) // 0 (기본값)
}
```

---

### **4. HashMap의 내부 구조 (버킷과 슬롯)**

- **버킷 (Bucket)**: 데이터를 저장하는 기본 단위
- **슬롯 (Slot)**: 각 버킷은 최대 **8개**의 슬롯을 가짐
- **해시 충돌**이 발생하면 **체이닝 (Chaining)** 방식으로 연결
- **슬롯**이 가득 차면 **새로운 버킷**이 생성되고 연결 리스트로 연결
- 모든 슬롯이 채워지면 **리사이징 (Resizing)** 발생

---

### **5. Hash 충돌의 원리**

- 서로 다른 **키(Key)**가 동일한 **해시 값**을 가질 때 발생
- 해시 충돌이 많아지면 성능 저하의 원인이 됨
- Go는 **SipHash**를 사용하여 보안성과 충돌 회피를 동시에 고려

---

### **6. 해시 충돌 확률**

- 32비트 해시 테이블: 충돌 확률 높음
- 64비트 해시 테이블: 충돌 확률 낮음 (사실상 무시할 수 있는 수준)

---

### **7. 32비트와 64비트 환경의 차이**

- 32비트: **4GB**의 메모리 주소 공간
- 64비트: **약 18.4엑사바이트**의 메모리 주소 공간
- Golang에서 **int**는 **플랫폼 의존적** (int32 또는 int64)

```go
package main

import (
    "fmt"
    "unsafe"
)

func main() {
    var i int
    fmt.Printf("int 크기: %d bytes\n", unsafe.Sizeof(i)) // 4 bytes (32bit) or 8 bytes (64bit)
}
```

---

### **8. 실무에서의 활용**

- **카운팅 (Counting)**
- **인덱스 생성 (Indexing)**
- **중복 체크 (Deduplication)**
- **캐싱 (Caching)**
- **키-값 기반의 데이터 매핑**
- **집계 처리 (Aggregation)**
