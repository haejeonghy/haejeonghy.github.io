# 커버링 인덱스
Created: 2025년 5월 27일 오전 9:56
Tags: Database, mysql
보관소: No
최종 편집 일시: 2026년 3월 14일 오전 1:08

커버링 인덱스(Covering Index)는 **쿼리를 실행할 때 필요한 모든 컬럼의 값을 인덱스만으로 가져올 수 있는 인덱스**를 말합니다. 즉, **테이블을 읽지 않고도 인덱스만으로 결과를 반환할 수 있기 때문에 성능이 매우 우수**합니다.

---

## **✅ 개념 정리**

### **📌 커버링 인덱스 (Covering Index)**

> 인덱스에 쿼리에서 사용하는 모든 컬럼이 포함되어 있어, 쿼리 실행 시 테이블의 실제 데이터를 조회하지 않아도 되는 인덱스
> 

---

## **✅ 왜 성능이 좋은가?**

- 일반 인덱스는 **인덱스를 통해 위치를 찾은 후 → 테이블에서 실제 데이터를 조회(=table lookup)**해야 함
- 커버링 인덱스는 **인덱스 자체에 필요한 모든 데이터가 들어 있으므로 table lookup을 생략**할 수 있음

---

## **✅ 예시 (MySQL 기준)**

### **🎯 예제 테이블**

```
CREATE TABLE user (
  id INT PRIMARY KEY,
  name VARCHAR(100),
  email VARCHAR(100),
  created_at DATETIME
);
```

### **🎯 커버링 인덱스 생성**

```
CREATE INDEX idx_name_email ON user(name, email);
```

### **🎯 커버링 인덱스가 되는 쿼리**

```
SELECT name, email FROM user WHERE name = '대장';
```

- 위 쿼리는 name을 조건으로 걸고, name과 email만 조회하므로 idx_name_email 인덱스 안에서 모두 해결됨
- → **테이블을 읽지 않음 → 성능 향상**

---

## **❗ 커버링 인덱스가 안 되는 경우**

```
SELECT name, email, created_at FROM user WHERE name = '대장';
```

- created_at 컬럼은 인덱스에 포함되어 있지 않기 때문에 테이블을 추가로 읽어야 함
- → **커버링 인덱스가 아님**

---

## **✅ 실제 확인 (MySQL)**

```
EXPLAIN SELECT name, email FROM user WHERE name = '대장';
```

- Extra 컬럼에 **Using index**라고 나오면 → 커버링 인덱스 사용 중
- Using where; Using index는 **조건 필터도 인덱스로 처리 + 테이블도 안 감**

---

## **✅ 정리**

| **구분** | **커버링 인덱스** | **일반 인덱스** |
| --- | --- | --- |
| Table Lookup | ❌ 없음 | ✅ 있음 |
| 성능 | 매우 빠름 | 상대적으로 느림 |
| 조건/조회 컬럼 | 모두 인덱스에 있어야 함 | 일부만 있어도 됨 |
| 용도 | 성능 최적화, 읽기 빠른 쿼리 | 일반 검색 최적화 |
