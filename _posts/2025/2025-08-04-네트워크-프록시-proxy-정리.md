# 네트워크 프록시(Proxy) 정리
Created: 2025년 8월 4일 오후 6:37

## ✅ 프록시란?

프록시는 네트워크에서 **클라이언트(사용자)와 서버 사이에 중간에 위치하는 서버**를 말합니다.

사용자의 요청을 **대신 처리하거나 전달하는 중계자** 역할을 합니다.

---

## 📌 핵심 개념

```
[사용자] ⇄ [프록시 서버] ⇄ [웹 서버]

```

- 사용자는 프록시에게 요청을 보냄
- 프록시는 실제 웹 서버에 요청을 보내고 응답을 다시 사용자에게 전달
- 사용자는 실제 서버와 직접 통신하지 않음

---

## 🧩 프록시가 하는 일

| 기능 | 설명 |
| --- | --- |
| 중계 | 요청을 서버에 대신 전달 |
| 캐싱 | 자주 조회되는 응답을 저장해 빠르게 응답 |
| 보안 | 사용자 IP를 숨기거나 암호화 지원 |
| 필터링 | 특정 웹사이트 차단, 트래픽 감시 등 |
| 속도 향상 | 중복 요청 제거, 압축, 캐시 사용 |

---

## 📦 프록시의 종류

### 1. 포워드 프록시 (Forward Proxy)

- 사용자가 외부 인터넷에 접근할 때 사용하는 프록시
- 회사, 학교, 내부망 등에서 자주 사용

```
[내 컴퓨터] ─▶ [프록시 서버] ─▶ [외부 웹사이트]

```

**특징**

- 사용자의 IP를 숨길 수 있음
- 외부 사이트에 직접 접근하지 않음
- 필터링 및 감시 가능

---

### 2. 리버스 프록시 (Reverse Proxy)

- 서버 앞단에서 클라이언트 요청을 대신 받아 처리하는 프록시

```
[사용자] ─▶ [리버스 프록시] ─▶ [내부 웹 서버]

```

**특징**

- 서버 보안 강화 (내부 서버 직접 노출 안 함)
- 로드밸런싱, 캐싱, SSL 처리 등 역할 수행

---

### 3. 투명 프록시 (Transparent Proxy)

- 사용자가 직접 설정하지 않아도 자동으로 적용되는 프록시
- 주로 공공기관, 기업망 등에서 감시/차단용으로 사용됨

---

## 💬 예시

### ▶ 회사에서 유튜브 접속 차단

- 모든 요청은 포워드 프록시를 통해 나감
- 프록시가 유튜브 요청을 감지하고 차단

### ▶ 네이버 접속 시 빠른 로딩

- 리버스 프록시가 썸네일 등의 자주 요청되는 응답을 캐싱
- 실제 서버까지 가지 않고도 빠른 응답 제공

---

## 🔧 주요 도구 및 시스템

| 도구/서비스 | 설명 |
| --- | --- |
| **Nginx** | 대표적인 리버스 프록시 서버 |
| **Squid** | 포워드 프록시 및 캐싱 프록시 |
| **Cloudflare** | 글로벌 CDN + 리버스 프록시 서비스 |
| **VPN / 프록시 앱** | 사용자 IP를 숨기고 다른 국가로 접속 가능 |

---

## 📌 요약 정리

| 구분 | 포워드 프록시 | 리버스 프록시 |
| --- | --- | --- |
| 위치 | 사용자 앞단 | 서버 앞단 |
| 목적 | 사용자 보호, 감시 | 서버 보호, 최적화 |
| 주요 기능 | 요청 필터링, IP 숨김 | 캐싱, SSL 종료, 로드밸런싱 |

---

## 🧭 연관 주제 (다음 내용으로 확장 가능)

- CDN (프록시 + 캐시 네트워크)
- Nginx 리버스 프록시 설정 예제
- Go 언어에서 프록시 서버 사용 (`http.Transport.Proxy`)
- 프록시 우회 방식 (VPN, SOCKS 등)
