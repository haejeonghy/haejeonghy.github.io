### **📝 파일 핸들 (File Handle) 정리**

---

### **1. 파일 핸들이란?**

- *파일 핸들 (File Handle)**은 **운영체제(OS)**에서 파일에 접근하기 위해 사용하는 **추상적인 참조값**입니다.
- 파일을 열거나 생성할 때, OS는 파일에 대한 **핸들**을 반환하여 이후 **읽기, 쓰기, 닫기**와 같은 작업을 할 수 있도록 합니다.
- 주로 **정수 (Integer)** 또는 **포인터 (Pointer)** 형태로 관리됩니다.

---

### **2. 파일 핸들의 역할**

- 파일의 **식별자 (Identifier)** 역할
- 파일의 **위치 (Offset)**, **모드 (읽기/쓰기)**, **권한 (Permission)** 등의 상태를 관리
- 운영체제의 **파일 시스템**을 통해 파일에 접근할 수 있는 **인터페이스** 제공

---

### **3. 주요 특징**

- 파일을 열면 **파일 핸들**이 할당되고, 파일을 닫으면 **해제**됨
- 동시에 열 수 있는 파일 핸들의 수는 **운영체제의 제한**을 받음
- 핸들을 적절히 닫지 않으면 **메모리 누수**나 **파일 잠금** 문제가 발생할 수 있음

---

### **4. Golang에서의 파일 핸들 사용 예시**

```
package main

import (
    "fmt"
    "os"
)

func main() {
    // 파일 열기
    file, err := os.Open("example.txt")
    if err != nil {
        fmt.Println("파일 열기 실패:", err)
        return
    }
    defer file.Close() // 파일 핸들 해제

    // 파일 핸들 정보 출력
    fmt.Printf("파일 핸들: %v\n", file)
}
```

---

### **5. 주요 메소드**

- **os.Open()**: 파일 열기 (읽기 전용)
- **os.Create()**: 파일 생성
- **os.OpenFile()**: 파일 열기 (읽기/쓰기 모드)
- **file.Close()**: 파일 닫기
- **file.Read()**: 파일 읽기
- **file.Write()**: 파일 쓰기

---

### **6. 파일 핸들을 잘못 다루면 생기는 문제**

- **메모리 누수 (Memory Leak)**
- **파일 잠금 (File Locking)**
- **데드락 (Deadlock)**
- **파일 시스템 성능 저하**
- **자원 부족 (Too many open files)**
