# 고루틴과 고 스케줄러 그리고 코루틴의 차이

## 1. 고루틴(Goroutine) 개념

고루틴은 **Go 언어**에서 제공하는 경량 스레드로, **`go` 키워드**를 사용하여 생성됩니다. 이는 기본적으로 **함수 또는 메서드를 비동기적으로 실행**하는 방법으로, 수천 개의 고루틴을 동시에 실행할 수 있을 정도로 매우 가볍고 효율적입니다.

### 📝 기본적인 사용 예시

```go
package main

import (
	"fmt"
	"time"
)

func printMessage(msg string) {
	for i := 0; i < 5; i++ {
		fmt.Println(msg, i)
		time.Sleep(100 * time.Millisecond)
	}
}

func main() {
	// 고루틴 시작
	go printMessage("고루틴")
	printMessage("메인")
}

```

**출력 예시** (실행할 때마다 달라질 수 있음)

```
메인 0
고루틴 0
메인 1
고루틴 1
메인 2
메인 3
고루틴 2
메인 4
고루틴 3
고루틴 4

```

### ✅ 주요 특징

- **경량성**: 일반 스레드보다 훨씬 적은 메모리와 리소스를 사용
- **자동 스케줄링**: Go 런타임이 직접 관리하여 복잡한 스레드 관리 불필요
- **비차단적 호출**: `go` 키워드를 통해 함수 호출이 비차단적으로 동작
- **스택 크기 동적 조절**: 초기 스택 크기가 작으며, 필요에 따라 자동으로 확장
- **병렬 처리 가능**: 다수의 CPU 코어를 활용하여 병렬로 실행 가능

## 2. Go 스케줄러 (GMP 모델)

고루틴의 실행은 **Go 스케줄러**에 의해 관리되며, 이는 **G (Goroutine), M (Machine), P (Processor)** 세 가지 요소로 구성된 **GMP 모델**을 기반으로 동작합니다.

### 🧩 구성 요소

- **G (Goroutine)**: 실제 실행되는 고루틴. 스택, 프로그램 카운터(PC), 상태 정보를 포함
- **M (Machine)**: 운영체제의 스레드. 실제로 고루틴을 실행하는 역할
- **P (Processor)**: 고루틴 실행에 필요한 리소스를 보유하며, 각 P는 하나의 M과 연결되어 고루틴을 실행

### 🔄 기본 동작 흐름

1. **고루틴 생성**
    - 고루틴이 생성되면 **G 구조체**로 생성되며, 대기열에 추가됩니다.
2. **P (Processor)**
    - 각 **P**는 하나의 **M**만 연결할 수 있으며, 고루틴의 실행 컨텍스트를 관리합니다.
3. **M (Machine)**
    - **P**와 연결된 **M**이 실제로 **G**를 실행합니다.
4. **스케줄링 (Work Stealing)**
    - 만약 **P**의 큐가 비어있으면 다른 **P**의 큐에서 **G**를 가져와 실행할 수 있습니다.

## 3. 카페를 비유로 이해하는 고루틴과 스케줄러

### ☕ 카페 시스템의 비유 요소 (GMP 모델)

1. **고루틴 (G, Goroutine)** - 주문서 📋
    - 각 손님의 음료 주문을 의미합니다.
    - 예시: 아메리카노 2잔, 라떼 1잔
2. **프로세서 (P, Processor)** - 커피 머신 ☕
    - 실제 음료를 만들 수 있는 작업 공간입니다.
    - 주방의 **작업대**나 **커피 머신**과 비슷한 역할입니다.
3. **머신 (M, Machine)** - 바리스타 👨‍🍳
    - 실제로 음료를 만드는 사람입니다.
    - 각 바리스타는 하나의 작업 공간(P)에서만 일할 수 있습니다.

### 📝 고루틴과 스케줄러 코드 예시 (카페 버전)

```go
package main

import (
	"fmt"
	"math/rand"
	"sync"
	"time"
)

func barista(order string, wg *sync.WaitGroup) {
	defer wg.Done()
	time.Sleep(time.Duration(rand.Intn(3)+1) * time.Second)
	fmt.Println("🍹 음료 준비 완료:", order)
}

func main() {
	orders := []string{"아메리카노", "카푸치노", "라떼", "에스프레소", "콜드브루"}
	var wg sync.WaitGroup

	fmt.Println("☕ 카페 주문 처리 시작")

	for _, order := range orders {
		wg.Add(1)
		go barista(order, &wg)
	}

	wg.Wait()
	fmt.Println("🛑 모든 음료 준비 완료!")
}

```

### 실행 결과 (예시)

```

🍹 음료 준비 완료: 아메리카노
🍹 음료 준비 완료: 카푸치노
🍹 음료 준비 완료: 라떼
🍹 음료 준비 완료: 에스프레소
🍹 음료 준비 완료: 콜드브루
🛑 모든 음료 준비 완료!

```

## 4. 고루틴과 코루틴의 차이점

### 📊 주요 차이점

| 특성 | 고루틴 (Goroutine) | 코루틴 (Coroutine) |
| --- | --- | --- |
| **동작 방식** | Go 런타임에서 관리 | 수동으로 양보 (협력적) |
| **스케줄링** | 선점형 (Preemptive) | 협력형 (Cooperative) |
| **운영체제 스레드** | 필요 시 스레드 사용 | 일반적으로 단일 스레드 |
| **병렬성** | 지원 (다중 코어 활용) | 일반적으로 단일 스레드 |
