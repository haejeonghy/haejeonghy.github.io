### 안드로이드 구성 요소

- 액티비티 : 사용자 인터페이스 화면 가지고 있는 요소 - 메인 스레드에 의해 실행됨
    - 생명주기 : onCreate() → onStart() → onResumed() → onPause() → onStop() → onDestroy()
- 뷰 : 보이는 요소 + 보이지 않는 요소
- 프래그먼트 : 화면의 작은 단위
    - onAttach() : 액티비티에 프래그먼트 추가
    - onDetach() : 액티비티에서 프래그먼트 삭제
    - onCreateView() : UI구성 View 반환
    - onActivityCreated() : 완전히 UI가 구성된 후 나머지 동작 구성
- 안드로이드 콜백 함수에서 lateinit 사용하기

```kotlin
class MainActivity : AppCompatActivity() {

		// 지연 초기화 선언
    private lateinit var mWelcomeTextView: TextView

		// 앱의 생성 주기(생성 시 호출되는 콜백 함수)
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
				// 지연 초기화
        mWelcomeTextView = findViewById(R.id.msgView) as TextView
    }
}

// 안드로이드 시스템 서비스 초기화 시에도 lateinit 사용 가능
class MainActivity : AppCompatActivity() {

		// 알람 시스템 서비스 선언
	  lateinit var alarmManager : AlarmManager

		// 앱의 생성 주기(생성 시 호출되는 콜백 함수)
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
				// 지연 초기화
        alarmManager = getSystemService(AlarmManager::class.java) // 지연 초기화 시점
    }
}
```

- 안드로이드에서 lazy 사용하여 UI 요소 초기화하기

```kotlin
class MainActivity : AppCompatActivity() {

    private val messageView: TextView by lazy {
				// messaveView 첫 접근에서 초기화
        findViewById(R.id.message_view) as TextView
    }
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        onSayHello()
    }
    
    fun onSayHello() {
        messageView.text = "Hello" // 이 시점에 초기화됨
    }
}
```

- 서비스 : 눈에 보이지 않는 백그라운드에서 실행됨
    - startService(), stopService() 등으로 서비스를 시작하거나 중단 가능
    - 실행하고자 하는 서비스가 또 다른 애플리케이션이나 시스템의 또 다른 프로세스에 있다면 다른 메모리 위치에 있는 원격서비스 구동 → 보통 xxxManager 역할을 하는 중간 매개체(proxy)의 서비스 → 통신을 위한 AIDL 정의 언어 → 원격 서비스에 명령
        - 이때 Parcel을 통해 전달됨
        - bindService(), unbindService() 사용
    - 생명주기 : startService() → onCreate() → onStartCommand() → stopService() → onDestroy() 호출
        
        bindService() → onBind() → unbindService() → onUnbind() 
        
- 브로드캐스트 리시버 : 안드로이드에서 발생하는 다양한 이벤트 및 정보를 받고 전달하는 요소
- 콘텐츠 프로바이더 (Content Provider) : 데이터를 관리하거나 다른 애플리케이션에 데이터를 제공하는 요소
    - SQLite, 파일, 웹 등을 저장 → URI 로 구분
- 인텐트 : 메시지 전달
    - 명시적 인텐트 : 특정 클래스 지정
    - 묵시적 인텐트 : 특정 데이터에 대해 수행할 액션 지정
- 핸들러와 메세지 큐
    - 안드로이드 앱 실행 시 메인스레드가 생성됨 → 메시지 큐와 루퍼가 자동으로 생성됨
    - 일반 스레드는 사용자에 의해 나중에 생성됨 → 일반 스레드에서 UI 직접 변경 코드 넣으면 안됨 → UI 변경 명령은 메시지 큐에 차례로 보내두었다가 하나씩 꺼내 액티비티의 UI 요소 변경하는 방법 사용 필요
    - 루퍼 : 메시지 큐 감시하면서 뭔가 오면 처리하는 반복 루틴
    - 이걸 코루틴으로 쉽게 해결 가능함
