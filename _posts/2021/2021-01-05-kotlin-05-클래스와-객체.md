### 코틀린에서 사용하는 용어와 다른 언어의 용어 비교

- 코틀린 : class ↔ 다른 언어 : 분류, 범주
- 코틀린 : property ↔ 다른 언어 : 속성, 변수, 필드, 데이터
- 코틀린 : method ↔ 다른 언어 : 함수, 동작, 행동
- 코틀린 : object ↔ 다른 언어 : 인스턴스

### 클래스

- 선언

```kotlin
class Bird {
	var name: String = "myBird"
	fun fly() = println("Fly")
}

fun main() {
	val coco = Bird()
	coco.name = "eunhyueok"
	coco.fly()
}
```

- 생성자 : 클래스를 통해 객체가 만들어질때 기본적으로 호출되는 함수

```kotlin
class 클래스 이름 constructor(필요한 매개변수..) {
	 constructor(필요한 매개변수..) {
	}
}

// 부 생성자 사용
class Bird {
	var name: String = "myBird"
	var wing: Int = 2
	fun fly() = println("Fly")
	constructor (name: String, wing: Int) {
		this.name = name
		this.wing = wing
	}
}
fun main() {
	val coco = Bird("eunyeok", 2)
	coco.name = "eunhyueok"
	coco.fly()
}

// 주 생성자 선언

class Bird constructor(name: String, wing: Int) { // constructor 단어 생략 가능
	var name: String = name
	var wing: Int = wing

	fun fly() = println("fly")
}

// 프로퍼티 포함 주 생성자 선언

class Bird constructor(var name: String, var wing: Int) {
	var name: String = $name
	var wing: Int = $wing

	fun fly() = println("fly")
}

// 초기화 블록 가지고 프로퍼티 포함 주 생성자 선언

class Bird constructor(var name: String, var wing: Int) {
	init {
		println("초기화")
	}

	fun fly() = println("fly")
}

// 프로퍼티 기본값 지정

class Bird (var name: String = "NONAME", var wing: Int = 2) {
	var name: String = $name
	var wing: Int = $wing

	fun fly() = println("fly")
}
```

### 상속과 다형성

- open : 클래스를 선언할 때 open 키워드를 명시 해야 파생 클래스에서 상속할 수 있음

```kotlin
open class 기반 클래스 이름 {

}

class 파생 클래스 이름 : 기반 클래스 이름 ( ) {

}

val someVal: Int
open class BaseClass(someArgs: Int)
class someClass(someArgs: Int) : BaseCLass(someArgs) // 클래스 상속 선언
class someClass : BaseClass { .. constructor() .. } // 부 생성자 사용 시 클래스 상속 선언
```

```kotlin
//메소드 상속시에도 open 언급 필요

open class Bird {
	open fun sing() {}
	final fun noSing() {
	} // 하위 클래스에서 재정의 할 수 없도록 막음 
}

class Lark() : Bird() {
	override fun sing() {
	}
}
```

- inner 클래스에서 바깥 클래스의 상위 클래스 참조 시 super  사용

```kotlin
open class Base {
	fun superF()
}

class Child : Base () {
	fun f()
	inner class Inside {
		Child().f()
		super@Child.superF()
	}
}
```

### 인터페이스

- 여러 인터페이스를 구현할 수 있으며 메서드나 프로퍼티명이 중복될 경우 앵글 브래킷 ( <> )을 사용해 구분할 수 있음

```kotlin
open class A {
	open fun f(){
	}
	fun a() = println("a")
}

interface B {
	fun f() = println("B interface f()")
}

class C : A(), B {
	super<A>.f()
	super<B>.f()
}
```

### 정보 은닉 캡슐화

- 가시성 지시자
    - private : 외부 접근 불가
    - public : 어디서든 접근 가능 (기본)
    - protected : 외부에서 접근할 수 없으나 하위 상속 요소에서는 가능
    - internal : 같은 정의의 모듈 내부에서는 접근 가능
