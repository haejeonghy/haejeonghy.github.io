### 필기

### 프로젝트 > 모듈 > 패키지 > 파일 (클래스)

- 대규모 프로젝트 진행 시 기능을 모듈로 분리하여 관리
- 별도로 패키지를 지정하지 않는 경우 default 패키지에 포함됨
- 파일에 클래스를 여러 개 정의할 경우 파일은 단순히 클래스를 묶는 역할만 함
- 패키지 : 같은 이름의 파일을 여러개 만들 경우 패키지가 달라야 중복 처리되지 않음

### 변수

- val : 최초로 지정한 변수의 값으로 초기화 후 더 이상 바꿀 수 없는 읽기 전용 변수
- var : 최초로 지정한 변수의 초깃값이 있더라도 값을 바꿀 수 있는 변수

```kotlin
val username: String = "Kildong"
// 코틀린이 자료형 추론하여 String으로 결정함
val username = "Kildong" 
var username // 오류! 자료형을 지정하지 않으면 사용 불가
```

### 자료형

- 참조형 자료형 사용
    - 기본형 : 가공되지 않은 순수한 자료형으로 프로그래밍 언어에 내장되어 있음
    - 참조형 : 객체를 생성하고 동적 메모리 영역에 데이터를 둔 다음 이것을 참조하는 자료형
- 부호가 있는 정수 자료형 : Long, Int, Short, Byte
- 부호가 없는 정수 자료형 : ULong, UInt, UShort, UByte
- 실수 자료형 : Double, Float
- 논리형 : Boolean
- 문자 자료형 : Char
- 문자열 자료형 : String

```kotlin
var a = 1
val s1 = "a is $a" // 변수 값이 $a에 할당됨 

// 표현식을 포함하는 경우 {} 중괄호 사용
val s2 = "a = ${a + 2}"
```

- typealias : 복잡한 구조의 자료형의 경우 별명 붙이기

```kotlin
typealias Username = String
// Username 은 String 의미
val user: Username = "Kildong" 
```

- ? : null 상태인 변수를 허용하기 위해 ? 붙여서 선언

```kotlin
var str1 : String? = "Hello Kotlin"
str1 = null
println("str1: $str1") // str1 = null
```

- 세이프 콜 (?.) : null이 할당되어 있을 가능성이 있는 변수를 검사하여 안전하게 호출하도록 함

```kotlin
// 변수 str1을 검사한 다음 null이 아니면
// str1의 멤버변수인 length에 접근하여 값을 읽어옴
println("str1: $str1 length: ${str1?.length})
```

- non-null (!!.) : 변수에 할당된 값이 null이 아님을 단정하므로 컴파일러가 null 검사 하지 않음

```kotlin
//NPE 발생
println("str1: $str1 length: ${str1!!.length}")
```

- 엘비스 연산자 (?:) : 변수가 null인지 아닌지 검사하여 null이 아니라면 왼쪽 식을 그대로 실행하고, null이라면 오른쪽 식을 실행함

```kotlin
// str1이 null이 아닌 경우 length, null인 경우 null 반환
${Str1?.length ?: -1}
```

- 스마트 캐스트 : 컴파일러가 자동으로 형변환 하는 자료형
    - Number : 저장되는 값에 따라 정수형, 실수형 등으로 자료형이 변환됨
- is : 자료형 검사

```kotlin
val num = 256
num is Int // true 
```

- Any : 코틀린의 최상위 기본 클래스로 어떤 자료형이라도 될 수 있는 특수한 자료형
- as : 형변환이 가능하지 않으면 예외 발생

```kotlin
// y가 null이 아니면 String으로 형 변환되어 x에 할당
val x : String = y as String

// null 가능성까지 고려하여 예외 발생을 피하려면 ? 사용
val x: String? = y as? String
```

### 연산자

- === : 2개 항의 참조 주소가 같으면 true, 다르면 false
- ! == : 2개 항의 참조 주소가 다르면 true, 같으면  false
